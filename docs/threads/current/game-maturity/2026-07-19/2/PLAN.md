## Compatibility invariants

1. Construction still prepares the default layout before the first player event without initializing renderer resources. Path and speed button positions, game-over rectangle geometry, update-helper order, integer arithmetic, and `_layout_size` assignment remain exact.
2. Unlock refresh invokes the existing progression calculation and setter once, blinks exactly the newly unlocked live buttons in ascending index order at the current `time_ms`, and then dynamically calls the public lock-state hook. Existing path-button identity and lock mutations remain canonical.
3. Purchase validation preserves unlocked-first rejection, live-list `.index` behavior and `ValueError` handling, sequential-price checks, progression debit timing, and the public unlock-refresh callback after a successful debit. Index validation retains `None`, negative, upper-bound, and Python `bool` behavior exactly.
4. `step_time` remains a public dynamic call to `increment_time`; it does not gain another clock, pause check, or scaling rule.
5. Surface sizing reads width then height exactly once each, uses `isinstance(value, (int, float))` so numeric subclasses and `bool` retain their current conversion behavior, and falls back independently to the current configured dimensions for other values.
6. Compatibility rendering re-reads surface size, prepares layout only on size mismatch through the public hook, constructs `GameRenderer` lazily only when both caller renderer and compatibility renderer are absent, reuses the stored renderer, resolves `draw` dynamically, and calls it with the exact `alpha` keyword. Rendering remains non-mutating outside layout preparation and compatibility renderer caching.
7. Game-over click handling returns early when nonterminal, tests restart before exit, reads the live rectangles at each condition, uses `Point.to_tuple()` at the same short-circuited points, and returns only `"restart"`, `"exit"`, or `None`.
8. Mouse dispatch performs one public hit test before branching. Down sets `is_mouse_down` before `isinstance(entity, Station)` and dynamically invokes path start. Up clears the flag before path completion/abort or button effects; `isinstance` path-button checks take precedence over `isinstance` speed-button checks, including subclass and multiple-inheritance cases; assigned path removal, locked purchase, speed action, and no-op branches preserve exact call order and partial state on exceptions.
9. Drag motion preserves live mouse/path flags, `isinstance` station extension versus temporary-point mutation, `isinstance` button hover versus all-button exit order, subclass and dual-class precedence, list replacement visibility, and public hook rebinding.
10. Keyboard dispatch handles only key-up events, toggles Space through direct live pause state, maps keys `1`, `2`, and `3` to speed multipliers `1`, `2`, and `4`, and dynamically calls the public speed hook at the same branches. Unknown events and keys remain no-ops.
11. Generic event dispatch preserves mouse-before-keyboard `isinstance` precedence and resolves the current facade-supplied event classes at their original checks. Hit testing preserves station order before button order and returns the exact first live object or `None`.
12. Pause and speed methods retain direct writable facade state. Speed actions preserve pause early return, unknown-action resume behavior, public setter ordering, and active-button truth rules while paused or at each multiplier.
13. Structured actions reject terminal state before action inspection, require `isinstance(action, dict)` so dictionary subclasses remain accepted, require `isinstance(action_type, str)` so string subclasses remain accepted, preserve exact `bool` versus `int` checks, default values, path-id precedence, public hook order, and return values for create, buy, remove, pause, resume, noop, invalid, and partially failing actions.
14. Public method signatures, direct state replacement, nested public dispatch, module-global rebinding, object identity, list/map identity, partial-state exceptions, and callable lifetime remain baseline behavior. No exception translation or rollback is added.
15. Human input remains dispatched before fixed updates; layout hitboxes remain prepared before input and are consumed rather than established by rendering. Structured and pixel environments continue to drive the same canonical simulation.
16. Protocol, action, reward, task, render, checkpoint, recursive, history, trainer, and manifest semantics remain unchanged. The new runtime module intentionally changes only content/source identity under existing drift rules.

## Size contract

The baseline `src/mediator.py` is 735 lines. The 19 cohesive algorithm bodies occupy 228 physical lines, but exact real facade methods have a mathematical delegation floor of about 43 lines before typed resolver wiring. Reaching 499 lines would therefore require stealing adjacent ownership or using magic/generated delegation. GM-03f instead targets `Mediator` at or below 600 lines with a hard ceiling of 625, requires `InputCoordinator` and every new Python test/support/verifier file below 500 lines, keeps every handwritten file below 1,000 lines, and records explicit final line-count accounting. Split any new Python file before it reaches 500 lines rather than treating the hard ceiling as its target. This is the practical interpretation of the parent under-500 target while preserving the facade and ownership constraints.
