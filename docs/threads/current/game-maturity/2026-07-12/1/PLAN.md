# GM-02 strategically spaced visual-history plan

Status: GM-02a implementation, adversarial review, and local validation green; Commit A staging in progress

Baseline: GM-01c Commit B `14050af71df5c6cad8035904da467959767f68bb`, green in GitHub Actions run `29181130841`

## Intent

Give fresh recurrent policies materially longer route context without pretending that a contiguous RGB stack alone supplies strategic memory. Preserve the existing eight newest 10 Hz decisions, add older route-state anchors, keep episode-long memory in the recurrent policy, and bind the exact temporal interpretation to every model artifact. The twelve-frame `decision-history-v1` candidate is not promoted until measured against two declared controls under the same rollout workload.

## Non-negotiable contracts

- The player-equivalent task remains one channel-first RGB frame with the current protocol and task fingerprints. Temporal history is a trainer-side observation transform with a separate descriptor and `historyFingerprint`; GM-02 must not rewrite the task identity merely because a model consumes more channels.
- A history descriptor is immutable and canonical. Callers supply only a versioned layout name and non-negative decision offsets ordered strictly oldest to newest and ending at zero; booleans, duplicates, and unsupported names fail closed. The descriptor schema fixes the offset unit to agent decisions, sample/channel order to oldest-to-newest sample-major channel-first RGB, pre-history fill to zeros, episode reset to clear-then-insert-initial, and terminal behavior to append/copy-before-auto-reset. These closed semantic literals serialize explicitly and cannot be replaced by arbitrary fingerprinted strings the runtime ignores. The fingerprint is SHA-256 over canonical JSON and the frame count is derived from the offsets.
- `decision-history-v1` uses oldest-to-newest offsets `[128, 64, 32, 16, 7, 6, 5, 4, 3, 2, 1, 0]`. At the default 10 Hz decision rate its anchors reach 12.8 wall-clock seconds at 1x and 51.2 simulated seconds at 4x while retaining the current eight recent frames. Before sufficient history exists, unavailable samples are zero-filled rather than duplicating the first frame.
- Training-manifest v2 stores both the descriptor and its independently recomputed `historyFingerprint`. It retains `frameStack` as a compatibility summary but rejects disagreement with the descriptor length. Manifest creation takes one descriptor and derives both summary and fingerprint rather than accepting contradictory sources. The existing v1 key set remains exact: parsing a genuine v1 manifest derives an internal contiguous descriptor with offsets `[N-1, ..., 0]`, and serializing that parsed v1 object reproduces v1 without injecting v2 keys. Fresh manifests are v2.
- Manifest-v2 parsing recomputes and verifies its stored fingerprint immediately. Compatibility validation also accepts an expected history fingerprint and reports history mismatch separately from task mismatch. Resume and evaluation reconstruct the history transform from the authenticated manifest; an explicit requested history must match the saved descriptor fingerprint, not only its channel count. Genuine manifest-v1 models with any positive contiguous stack, including 1, 4, 8, and another arbitrary N, remain loadable and keep their old SB3 stacking semantics.
- GM-02a and GM-02b still execute SB3 `VecFrameStack`, so their runtime boundary accepts only a descriptor exactly equal to contiguous `[frameStack-1, ..., 0]` and rejects any other otherwise-valid v2 layout before environment/model construction. Those phases emit only the actual contiguous descriptor. GM-02c removes this temporary guard only when descriptor-driven train, resume, and evaluation construction all land together.
- The dependency-light descriptor lives in `src/rl/history.py`; the SB3/Gymnasium wrapper lives separately in `src/rl/temporal_history.py` and is imported locally inside `build_vector_env()` after dependency validation. Both modules enter the closed training-source fingerprint allowlist while the environment-content fingerprint remains unaffected by trainer-only history code.
- The temporal wrapper operates on channel-first `uint8` box observations and preserves dtype/bounds. It owns one bounded circular ring, write cursor, and maximum-valid-age counter per vector slot; it never uses `np.roll`, copies a full parent ring per decision, or shares episode history between workers. Unsupported observation spaces or malformed observations fail closed.
- Wrapper order is `DummyVecEnv` or spawned `SubprocVecEnv`, then `VecMonitor`, then temporal history. On reset, every slot is cleared and only the new initial observation occupies offset zero. On an ordinary step, the returned stack is sampled after appending that step's observation. On an auto-resetting terminal/truncated step, the wrapper validates the single-frame terminal observation, appends it to the ending ring, assembles and owns a copy of the terminal stack, and only then clears/reuses that slot for the returned replacement episode's initial observation. Missing or wrong-shape/dtype terminal observations fail closed; non-done slots, rewards, dones, and `TimeLimit.truncated` pass through unchanged. Returned stacks and terminal stacks must not alias storage that a later step mutates.
- Time-limit terminal observations remain available for Stable-Baselines3 timeout bootstrapping, and recurrent `episode_start` masks continue to come from `dones`. Training, evaluation, save/resume, and both `DummyVecEnv` and spawn-based `SubprocVecEnv` use the same descriptor.
- History storage stays `uint8`; no full-ring float conversion or per-step unbounded allocation is allowed. Default promotion is conditional on measured process-tree peak RSS and throughput, not only analytical tensor size.

## Acceptance

- Manifests cannot silently reinterpret equal-channel temporal layouts; v2 descriptor and fingerprint tampering fails closed while genuine v1 contiguous artifacts remain exact and usable.
- Synthetic and real vector tests prove chronology, zero pre-history, staggered reset isolation, terminal/truncated stacks, timeout bootstrap, recurrent masks, and spawn behavior.
- Fresh training observes more than eight strategically spaced frames only after the matched resource profile proves the selected default safe. Evaluation and resume reconstruct precisely the saved layout.
- The single-frame task fingerprint remains unchanged; trainer-source fingerprint changes intentionally; environment-content fingerprint changes only if separately justified. All local applicable gates and pinned remote A/B gates pass, with limitations reported precisely.
