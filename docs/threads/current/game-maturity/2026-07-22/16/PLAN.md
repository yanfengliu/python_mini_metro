## Design (D-030)

- New main-only `src/audio.py` (< 500), holding ALL its own constants (it imports `pygame`/`numpy` only — NOT `config`, resolving the plan-review MINOR-6 wording and keeping the high-risk `config.py` untouched):
  - Tone constants defined in-module: a fixed sample rate (`_SAMPLE_RATE = 44100`) and per-event `(frequency_hz, duration_ms)` specs for the five events `delivery`, `path_unlock`, `station_unlock`, `game_over`, `snap` — distinct, short, unobtrusive. A pure `_generate_tone(spec) -> numpy int16 ndarray` builds a deterministic MONO waveform (fixed sine formula + a short linear attack/decay envelope to avoid clicks); given the fixed inputs the bytes are byte-stable and device-independent (mirroring `menu_screens._font`'s bundled-deterministic pattern, no external file). A `_shape_for_mixer(mono, channels) -> ndarray` duplicates the mono column to the mixer's channel count.
  - `ProceduralAudio`: initialized with the mixer's channel count read from `pygame.mixer.get_init()`, it builds one `pygame.mixer.Sound` per event via `pygame.sndarray.make_sound(_shape_for_mixer(tone, channels))` (review MAJOR-2 — the mixer inits STEREO, so a 1-D array raises `ValueError`; the sound array MUST match the channel count), and `play(event, master_percent, sfx_percent)` sets the sound volume to `(master/100) * (sfx/100)` via `Sound.set_volume` and plays it, guarding a `None` channel and swallowing any play error (best-effort). Holds no gameplay reference.
  - `NullAudio`: a no-op backend (`play(...)` does nothing), for headless/no-device and graceful degradation.
  - `create_audio() -> backend`: initializes `pygame.mixer.init(frequency=_SAMPLE_RATE, size=-16)` and constructs `ProceduralAudio` (building every sound) ALL inside one `try/except (pygame.error, Exception)` (review MINOR-3 — the try must cover the `make_sound` shape/build failures, not just `init`, so a post-init failure also degrades); returns `NullAudio` on any failure, so audio-init failure never blocks gameplay or tests. Patchable at the `main` layer as `main.create_audio`.
  - `diff_and_play(host, snapshot, backend, master_percent, sfx_percent) -> new_snapshot` (review MINOR-2): a PURE, duck-typed differ that reads `host.deliveries`/`host.unlocked_num_paths`/`host.unlocked_num_stations`/`host.is_game_over`/the snap-blip sum off the host, compares to the prior `snapshot`, plays one tone per newly-occurred event through `backend`, and returns the new snapshot. It imports no gameplay (reads attributes only — the isolation scans check imports, not attribute access), so it is unit-testable directly with a `SimpleNamespace` host and keeps `main`'s wiring to a few lines.
  - No gameplay/rendering/save import; `audio` joins the isolation set and lives OUTSIDE `rendering/` (which `rl/player_env.py` imports transitively), so `main` is its sole importer.
- `src/main.py` constructs the backend once (`create_audio()`, patchable) plus its OWN `previous_audio_session` reference and an initial counter snapshot — it must NOT reuse the loop's `previous_session`, which is already advanced to `session` at the top of the render block BEFORE the post-advance hook, so reusing it would never reset and a Continue-loaded mediator's non-zero counters would fire a spurious `delivery`+`path_unlock`+`station_unlock` tone burst (review MAJOR-1; `_continue_game` swaps in a loaded mediator with non-zero counters). At the post-advance hook (right after `controller.reconcile_game_over()`, reading the POST-reconcile `state` so the promotion-frame `game_over` tone is allowed — the review NIT), the consumer: (1) if `controller.session is not previous_audio_session`, re-initializes the snapshot to the current mediator's values and updates `previous_audio_session` (so New Game / Restart / Continue never replay a stored delta), then (2) while `state in (PLAYING, PAUSE_MENU, GAME_OVER)`, calls `audio.diff_and_play(controller.mediator, snapshot, backend, master, sfx)` — one tone per newly-occurred `deliveries` / `unlocked_num_paths` / `unlocked_num_stations` / `is_game_over` / snap-sum delta — reading the live `controller.current_settings.master_volume` / `sfx_volume`. The snap signal is a best-effort sum-increase on `sum(len(s.snap_blips) for s in mediator.all_stations)` (review MINOR-1 — `prune_visual_effects` shrinks the list every active tick, so a snap coinciding with a blip expiry can net a non-positive delta and be missed; acceptable for a cosmetic click, and it adds no monotonic Station counter, which would bump the frozen save-v1 schema). Volumes are SFX; `music_volume` has no consumer until a future music track.
- Optional inert seam: `main.run_game` builds the audio backend and passes a small `audio` object into the consumer; a `max_frames`/headless or test run injects `NullAudio` (or the factory yields it when no device), so nothing is audible and nothing crashes. The `AppController` is unchanged (audio is a pure loop-level consumer, not a screen), keeping the controller surface stable.
- Isolation: `env.py`, `agent_play.py`, `recursive_playtest.py`, `recursive_checkpoint.py`, and `rl/*` gain no `audio` import. `test_gm07b_save_determinism.py`'s `SAVE_MODULE_NAMES` set gains `"audio"` (a persistence-set name that now also carries a non-save runtime module — noted in a comment, review MINOR-5), and the GM-08a mirror in `test_gm08a_settings_render.py`, which hardcodes the literal `"settings"`, is generalized to forbid each of `{"settings", "audio"}`. Both scans are direct-import only, which suffices because `main` is the sole importer (grep-confirmed); the load-bearing guarantee is simply that only `main` imports `audio`. The consumer fires only in `main`'s loop, so headless and RL play never construct a mixer.

## Acceptance

- Finishing a delivery, purchasing a line, unlocking a station, and reaching game over each play a distinct short tone at the settings-scaled SFX volume; snapping a line endpoint plays a best-effort tone (a snap coinciding with a blip expiry may be missed — a cosmetic limitation that avoids a schema-bumping monotonic counter); tone generation is deterministic and byte-stable.
- Audio initialization failure or a missing device degrades to `NullAudio` and can never crash or block gameplay or tests; the full suite and headless/RL play construct no mixer and stay silent.
- No `Mediator`/`GameSession`/`rendering` change, no schema/observation/checkpoint/frozen-artifact change; `audio` is named in both isolation scans and imported only by `main`; all budgets and gates hold.
