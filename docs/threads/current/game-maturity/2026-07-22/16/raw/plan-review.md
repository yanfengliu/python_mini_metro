# GM-08b PLAN — Adversarial Review (harness lane)

Reviewed against live code at `main`. The architecture (main-only `audio.py`, per-frame counter differ, `NullAudio` fail-safe, no mediator/schema change) is sound and correctly isolated, but two MAJOR pre-code gaps would ship real defects as written, plus several MINORs.

## Findings

**[MAJOR-1] Session-change reset cannot reuse main's `previous_session` at the chosen insertion point → spurious tone burst on Continue.** `main.py:269` executes `previous_session = session` BEFORE the reconcile hook at `main.py:277`, so at 277 `previous_session is session` is always True and a differ reusing that variable never resets. On Continue (`app_controller._continue_game`, controller.py:173-188 / :186) the loaded mediator's counters jump above the title baseline snapshot `(0,1,3)` to e.g. `(30,2,5)` while state is PLAYING → spurious `delivery`+`path_unlock`+`station_unlock` tone burst. New Game/Restart deltas are negative (no burst), so the reset's only load-bearing case (Continue) is exactly the one that breaks. The differ must run after reconcile to observe post-promotion state, so it cannot move before line 269. Amendment: the audio consumer owns its OWN `previous_audio_session` reference bundled with the counter snapshot, reset to the current mediator's values whenever `controller.session` changes; forbid reusing main's `previous_session`.

**[MAJOR-2] Stereo mixer shape: `make_sound` on a 1-D array raises, and the fail-safe would hide it as permanent silence.** On pygame-ce 2.5.7 (and the dummy driver) `pygame.mixer.get_init() == (44100,-16,2)` — stereo — and `pygame.sndarray.make_sound(mono_1D_int16)` raises `ValueError: Array must be 2-dimensional for stereo mixer`. `_generate_tone` naturally returns 1-D (bytes pinned at PLAN.md:16). If `create_audio` wraps ProceduralAudio construction in `try/except (pygame.error, Exception)`, the ValueError is caught → `NullAudio` → the game is silent on every run while every "degrades / never crashes" test passes and the byte-stable mono test never exercises `make_sound`. Amendment: (a) pin `pygame.mixer.init(frequency=SAMPLE_RATE, size=-16, channels=...)`; either init `channels=1` + pass 1-D, or reshape mono→`(n, get_init()[2])` before `make_sound`; (b) the `SDL_AUDIODRIVER=dummy` test MUST assert `create_audio()` returns a real `ProceduralAudio` with all five sounds built (not degraded `NullAudio`) and each plays.

**[MINOR-1 → borderline MAJOR] Snap signal is non-monotonic (pruned every tick), so "sum-increase" silently misses snaps.** `station.prune_visual_effects` (station.py:87-88) shrinks `snap_blips` to only-active (<400ms) and runs unconditionally per active tick at `passenger_flow.py:159` (test_mediator_simulation.py:105-112). So `sum(len(snap_blips))` rises on a snap but falls 400ms later; a snap coinciding with an expiry nets ≤0 and is missed. Cannot double-fire (sum only rises via `start_snap_blip`, path_lifecycle.py:389/394/430). A new monotonic per-station snap counter is forbidden — `snap_blips` is persisted (save_game.py:76 "snapBlips", restored save_load.py:86) and a new Station field bumps frozen save-v1 (D-026). Amendment: keep sum-increase but downgrade the acceptance wording to best-effort (note the coincident-expiry miss), or drop snap. Do NOT add a Station counter.

**[MINOR-2] The differ test cannot use a bare `MagicMock` mediator; extract a pure differ into `audio.py`.** `test_gm08a_settings_main` patches `main.Mediator` (a MagicMock), which makes `sum(len(s.snap_blips))` and integer comparisons meaningless. Amendment: extract `diff_and_play(host, snapshot, backend, master, sfx) -> snapshot` into `audio.py` (reads attributes, imports no gameplay — isolation-clean), unit-test it directly with a `SimpleNamespace` host; main keeps a ~2-line wiring call.

**[MINOR-3] `create_audio` try scope + real-mixer in the dev suite.** (a) The try/except must wrap ProceduralAudio construction (the `make_sound` calls), not just `mixer.init()`. (b) `play()` best-effort + guard `Sound.play()` returning `None` (no free channel). (c) Critical: `audio.py` imports its OWN pygame, so `patch("main.pygame")` does NOT stop a real `mixer.init()`. Every loop-driven differ test MUST patch `main.create_audio` to inject `NullAudio`/a fake, or it opens a real mixer on this Windows dev box. Only the dedicated dummy test may call the real factory.

**[MINOR-4] Test hygiene for the real-mixer (`dummy`) test.** The full suite doesn't set `SDL_AUDIODRIVER` and cycles `pygame.init/quit`. `os.environ["SDL_AUDIODRIVER"]="dummy"` only affects a fresh init; the dummy test must `pygame.mixer.quit()` first, set the env, init fresh, and `mixer.quit()` in tearDown so it never leaks mixer state into later tests. Ensure the factory is invoked after `pygame.init()` (main.py:151).

**[MINOR-5] Isolation-scan extension is coarser than "extend to name audio".** `SAVE_MODULE_NAMES` (test_gm07b_save_determinism.py:29-36) adding `"audio"` works but the set name / "ALL FOUR save modules" comment (:293) become misnomers — note the drift. The GM-08a mirror hardcodes the literal `"settings"` (test_gm08a_settings_render.py:256), so extension means forbidding each of `{"settings","audio"}`. Both scans are direct-import only; the real guarantee is "only main imports audio". `rl/player_env.py:16` imports `rendering.game_renderer`, so keeping audio OUTSIDE `rendering/` (plan: `src/audio.py`) is correct and necessary.

**[MINOR-6] Plan internal inconsistency: does `audio.py` import `config`?** PLAN.md:15-16 says it HOLDS the constants; PLAN.md:20 / D-030 say it IMPORTS `config` tone constants. Resolve. `config.py` is import-safe (imports only `geometry.type`, pygame-free) but is on the AGENTS.md locally-high-risk list; putting tone specs in `audio.py` avoids touching `config.py`. Pick one.

**[NIT]** The differ must gate on `controller.state` re-read AFTER `reconcile_game_over` (main.py:278), not the stale `state` from main.py:260, so the `game_over` tone is allowed on the promotion frame. **[NIT]** main.py is 335 lines (still far under 500 with the differ inlined).
