# GM-07b research — determinism, versioning precedent, and test oracles (synthesis of the verbatim lane report)

- Seeded-trajectory determinism: `SimulationContext` (simulation_context.py:13-21) is the single randomness source — `SeedSequence(seed).spawn(2)` feeding `random.Random` and PCG64; every tick-path consumer routes through it (station pool get_entity.py:54,78; colors mediator.py:324-336; spawn interval passenger_flow.py:97; destination choice :127; route shuffle mediator.py:763). Module-global fallbacks exist only for `context is None` and are proven unused by `test_env_reset_and_step_do_not_change_host_global_rngs` (test_simulation_context.py:59-71). No wall-clock feeds decisions. With both streams captured and restored, save-then-load reproduces the identical subsequent trajectory.
- Hash/order stability: CLEAN. BFS iterates the insertion-ordered `_OrderedNodeSet` (node.py:15,28-45; graph_algo.py:55) built deterministically; `node.paths` (a plain set) never feeds ordering (intersection truthiness only; checkpoint output sorted); shape-type dedup preserves list order (passenger_flow.py:72); route tie-breaks use strict `<` over RNG-shuffled input; every `id()` usage is order-independent membership or feeds deterministic-list-ordered index maps. `PYTHONHASHSEED=0` in recursive runs is belt-and-suspenders.
- Versioning pattern to follow: `schemaVersion` int top-level; `<NAME>_SCHEMA_VERSION_V1..VN` constants with a supported set; `type(value) is not ...` exact-type checks rejecting bool-as-int; `_exact_keys(required, optional)` rejecting unknown and missing; pinned literal contract strings; forward versions rejected outright, backward versions upgraded on a safe copy; live-only sets rejected in persisted documents at every version (recursive_contract.py, recursive_checkpoint_schema.py).
- Frozen-fixture recipe: `scripts/fixtures/*.json`; bytes = `json.dumps(value, allow_nan=False, ensure_ascii=True, separators=(",",":"), sort_keys=True).encode("ascii") + b"\n"`; pinned by SHA-256 and byte length inside the test file; no CR; no `.gitattributes` entries for fixtures (LF discipline by writing bytes).
- Reusable oracles: canonical_checkpoint (UUID-free, embeds both RNG states at recursive_checkpoint.py:485-488) as the state-equality oracle; the lockstep two-env trajectory pattern (test_simulation_context.py:48-57); the both-streams-tracked oracle (:73-86); array-aware numpy state comparison (:18-23); the in-memory RNG capture/restore idioms; digest (canonical bytes + sha256); the fresh-process `PYTHONHASHSEED=0` cross-run byte-identity pattern (test_gm06c_historical_compatibility.py:133-189).
- Public-ID oracle shape: pre-save `observe()["structured"]["paths"][k]["id"]` → save → load → `apply_action({"type": "remove_path", "path_id": id})` returns True. ID consumers enumerated (input_coordinator.py:452-489; fleet_input.py:27-58; mediator.py:473-510,784-785); observation ID surfaces enumerated (env.py:164-244).
- Atomic write: `write_json_atomic` (rl/artifacts.py:58-83) — mkstemp in destination dir → write → flush → fsync → `os.replace` → finally-unlink; note its UTF-8/`ensure_ascii=False` divergence from the ASCII fixture recipe (byte-pinned fixtures force the ASCII path).
- Isolation: AppController owns the sole reconstruction path (`build_game` from main.py:37-42); env/agent/recursive drive Mediator directly and must gain no save behavior; the global-RNG isolation oracle must stay green.

Risks/open decisions carried into the plan: no load precedent exists anywhere (the three restore_* helpers are in-memory rollbacks over live objects, not JSON deserialization — the largest gap); the UUID-free equality oracle cannot catch ID drift, so a separate ID-preservation oracle is mandatory; ASCII-canonical vs UTF-8 encoding must be chosen (fixtures force ASCII); no save directory exists (`output/` is ignored and RL-owned — flagged rather than defaulted); live-only/transient state needs an explicit exclusion policy; reusing checkpoint validators would reject persisted ID maps via `_reject_entity_ids`, so the save schema needs its own validator family.
