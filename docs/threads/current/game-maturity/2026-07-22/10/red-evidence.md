## Implementation-phase oracle correction

The implementation exposed a design contradiction in two red oracles, corrected as a review-driven contract amendment: byte-exact fixture REGENERATION and fresh-process fresh-BUILD byte identity are impossible by design for an ID-bearing schema (entity IDs are shortuuid-minted independently of the game seed — the research's own load-bearing fact; the checkpoint fixtures regenerate byte-exactly only because they are UUID-free). Both tests were re-aimed at strictly stronger honest oracles: cross-process load→re-save byte idempotence over the frozen fixture (pins the serializer, canonical emission, and hash-seed independence — passed on first run) and regeneration equivalence through the UUID-free checkpoint oracle plus in-process load→re-save byte idempotence. The fixture byte/digest pins on the frozen artifact are unchanged. Final green: four modules 48/48; full suite 1129/0 (12 skips); guarded `npm test` 249/0 after one isolated-verified recurrence of the known pre-existing Windows EPERM ledger-lock flake (9/9 in isolation, exactly as recorded in GM-06c's delivery).

## Review-fix amendment (post-adversarial-review, same unit)

Both review lanes (`raw/codex.md`, `raw/harness-review.md`) returned NOT CLEAN with one converging blocker: the plan's "`_station_service_action` is derived/transient, reconcile-then-re-apply at load" design was wrong for reachable states — `move_passengers` is sequential over `host.metros`, so a later metro can consume the passenger an earlier metro's cache is bound to, leaving a legitimately stale cache at the boundary that the loader could not reconstruct (rejected loads) or silently mis-attributed (boarding progress re-bound to a freshly derived action). Amendment: schema v1 gained a nullable per-metro `serviceAction` record (`{"kind", "passengerId"}` mirroring the live tuple) with fail-closed invariants (bound ⇒ at-station, speed 0, `stop == interval - progress`, `0 <= progress < interval`; null ⇒ timers `(0, 0)`); the serializer emits the cache verbatim and the loader restores it verbatim — never re-deriving — so a persisted null with a derivable action correctly replays the live stale-reset semantics. The load-time reconcile+re-apply machinery was removed and the fixture refrozen (15442 bytes, sha256 `d34736a6…deacd1e`).
