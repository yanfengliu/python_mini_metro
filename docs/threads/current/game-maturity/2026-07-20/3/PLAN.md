## Gesture contract

- Converted primary-pointer down on an assigned path button arms that exact live path without changing its topology, riders, fleet, button mapping, color, RNG, clocks, or simulation behavior. Down on locked or unassigned buttons does not arm redraw.
- Arming is allowed only when `is_creating_path` is false, `path_being_created` is `None`, no redraw is already armed, and the down begins from mouse-up state. Repeated/multi-button downs never switch targets or start creation. If malformed state contains both creation and redraw, creation retains precedence and redraw clears fail-closed; real-event interleaving tests pin this boundary.
- While held, entering the first station starts a facade-owned off-live draft. Later unique stations extend it. Consecutive repeats are ignored. Revisiting the first station after at least two unique stations closes a loop; moving away and then visiting a new unique station reopens the draft before extending it. Revisiting any other captured station marks the draft invalid rather than silently reordering it.
- Release precedence is exact. With no redraw armed, mouse-up keeps the complete historical release-target behavior. With redraw armed but zero captured stations, mouse-up first clears redraw and then delegates that same complete matrix: assigned button deletes its release target (down on A/up on A deletes A; down on A/up on B deletes B), locked path button attempts purchase, speed button applies its action, and station or empty-space release has no effect. Once at least one station has been captured, a release whose resulting draft is short, invalid, on any button, or off-station cancels with no deletion, purchase, speed action, or replacement hook. A station release that produces a valid linear or loop draft of at least two stations is the only branch that commits.
- The valid station-release branch includes that station, clears transient redraw and hover state before calling the current public `Mediator.replace_path(path, station_indices, loop)` hook exactly once, and relies on the canonical GM-05a transaction for all identity, metro, passenger, RNG, rollback, and unsafe-edit behavior. GM-05b first extends replacement preflight, using safe absent-field lookup for historical/direct hosts, so every structured/API replacement rejects while a redraw is armed; the UI then clears its own draft before the one intended commit call, and the field remains clear if a rebound hook raises. An arbitrary rebound hook is outside GM-05a's declared rollback surface: the input layer guarantees gesture/hover cleanup but does not claim to reverse unrelated mutations made by that hook.
- A valid but unsafe replacement returns to the unchanged old line through GM-05a's fail-closed contract. Plain assigned-button click deletion, including the historical bare mouse-up path, remains compatible under the release matrix above.
- Normal station-origin path creation, locked line purchase, speed buttons, hover/cross behavior, hit-test precedence, and structured `replace_path` remain available and retain their established results. `convert_pygame_event` still does not persist the pygame button number; tests send `button=1`, while broader button-model changes remain outside GM-05b.
- Arming redraw calls the established button-exit behavior for all buttons so the hover-only delete cross cannot remain stale after the cursor leaves under held motion. Success, cancellation, invalid/unsafe rejection, and raised-hook cleanup leave every button hover/cross state consistent with the release position; bare-up deletion and ordinary mouse-up hover semantics remain compatible.

The redraw draft is transient player-interface state owned directly by `Mediator` and manipulated by the stateless `InputCoordinator`; no controller retains a facade or gesture across hosts. The current recursive scenario/checkpoint contract operates on completed structured actions and already excludes low-level pointer-down state, while `PlayerPixelEnv` owns its cursor/button lifecycle separately. GM-05b therefore adds no persisted schema field and specifically adds no optional or additive field to checkpoint v2: direct tests pin every transient field during the gesture, existing pixel observations pin the selected/pressed preview, and completed or canceled gestures must clear the draft and produce the same canonical checkpoint as the equivalent structured action or unchanged baseline. If the draft ever enters live paths, graphs, plans, RNG, structured arrays, simulation, or a mid-gesture resume contract, that is a real new checkpoint schema plus legacy normalization/replay migration and is outside this plan.

## Preview and feedback contract

- The completed live line remains operational and unchanged throughout the drag. The renderer derives a separate immutable preview from the draft's station objects, selected line color, live centered lane order, loop state, and temporary pointer position without constructing a gameplay `Path`, allocating public IDs, rebuilding logical segments, or touching the network cache key.
- The selected path button receives a deterministic outline and suppresses its delete cross while armed. An invalid duplicate draft receives deterministic error styling. The preview is drawn above the cached old network and below stations/metros/buttons so the planned route is visible without obscuring entity state.
- Preview construction and drawing are observational: they do not write path, station, draft, renderer input, RNG, clock, snap-blip, or checkpoint-visible gameplay state. Repeated identical software renders are byte-identical and leave the one-entry static network cache bounded.
- The preview follows the same centered lane-offset and junction-padding geometry and shared two-pass halo-then-color rasterizer as completed network rendering, including linear corners, loop closing padding, reversed edges, and shared-lane order. An isolated snapped preview is byte-identical to its committed candidate surface. Above-old-network selection overlay is intentionally not globally byte-identical at crossings because its halo emphasizes the active draft until commit; that transition is documented and tested rather than mistaken for parity.
- Static and preview caches are separate and bounded. Preview keys/artifacts contain only immutable numeric/string/signature data and surfaces, never Path, Station, Gesture, or host references. Selection, station snaps, invalidation, and pointer-only motion leave the static `cache_rebuild_count` exact; a topology-changing commit increments it once, while GM-05a's successful exact same-route no-op leaves it unchanged. A real-gesture same-route regression pins that zero-rebuild result.

## Interpolation continuity contract

The reproduced defect is reachable in the real main loop: input dispatch precedes `GameSession.advance`, and a 60 Hz frame may execute zero fixed steps. A replacement can therefore preserve the live metro pose and path ID while stale renderer snapshots still name an old segment index that now means another edge, producing an immediate one-frame teleport.

`MetroSnapshot` will retain the exact bounded current-segment object and an immutable physical-transition key captured from its containing path. `MetroInterpolator.pose_for` may reuse its cached current snapshot directly only when the live path still contains that exact segment at the recorded index, and it validates a previous snapshot against its own path index before projection. On a topology mismatch with unchanged current physical position, matching old previous/current segment identity proves only that both snapshots shared one old segment; rebase additionally requires equal normalized old-to-live transition keys. A path-segment key contains segment kind, the unordered station-identity pair, and unordered endpoint geometry. A padding key contains segment kind, both adjacent unordered station-identity pairs normalized as a set, and unordered endpoint geometry, including loop-closing adjacency; it is therefore invariant under whole-route reversal but distinguishes retained arrival A-B followed by changed outgoing B-C versus B-D. Missing or unequal padding context always falls back, which explicitly excludes a stopped metro whose already-advanced outgoing padding is rerouted or becomes a terminus. When compatibility holds, the renderer rebases both snapshots to the live segment index and applies the old-current-to-live direction flip to each independently, preserving prior/current stopped flags, progress, heading, turnaround, and the exact sub-tick pose. If snapshots span transitions, segment semantics differ, or any other precondition fails, the renderer projects the live metro and never animates through a removed or reinterpreted edge. After the next observed step, ordinary interpolation resumes, including genuine in-step segment transitions. The renderer retains at most the same previous/current snapshot maps; weak-reference tests prove old segment objects release after map rotation or clear.
