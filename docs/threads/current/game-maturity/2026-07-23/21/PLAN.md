## Design (the recon's low-risk steer)

The load-bearing choice: **`available_tunnels` is DERIVED from live-path crossings, exactly like `available_locomotives`** — NOT a stored counter. So consumption is automatic, and rollback/refund come FREE with ZERO new snapshot fields (the intricate `replace_path`/`remove_path` transactions and their `snapshot_state`/`restore_state` are UNTOUCHED — restoring the geometry restores the count). The route-edit gate is a READ-ONLY check of the candidate's crossing count.

- **Crossing geometry** — a new pure `src/crossings.py` (import-light: `geometry.point` + plain rect tuples, NO shapely, NO pygame/entity): `segment_crosses_band(start, end, band) -> Point | None` (a pure segment-vs-axis-aligned-rect test — Liang-Barsky/slab — returning the ENTRY point the renderer needs, or None), and `path_crossings(station_positions, is_looped, rivers) -> tuple[Point, ...]` that walks a path's **CENTERLINE** (consecutive `station.position` pairs + the loop-closure pair) and collects every river-band crossing. CENTERLINE, not the lane-offset `PathSegment` geometry (which is `path_order`-dependent and `round()`-quantized → non-deterministic w.r.t. the logical route — recon §7). Deterministic and stable.
- **The tunnel budget on the map** — `MapDefinition` gains `tunnel_budget: int | None = None` (per-map, not global `config`: CLASSIC has no river so `None` = unbounded/no constraint; `RIVER` gets a small finite budget — proposed 3, tunable — so the river is a meaningful constraint but a connected network is buildable). `Mediator` reads `self.num_tunnels = self.map_definition.tunnel_budget` (mirroring `num_metros` at 116-117).
- **Derived counts on the Mediator** — `consumed_tunnels` property = `sum(len(path_crossings(p.stations→positions, p.is_looped, self.map_definition.rivers)) for p in self.paths)`; `available_tunnels` property = `None` when `num_tunnels is None` (unbounded), else `max(0, num_tunnels - consumed_tunnels)`. Mirrors `available_locomotives`; no stored state, so every existing rollback restores it for free.
- **Route-edit enforcement (read-only gate)** — in `path_replacement` preflight (the identity-preserving reroute at `path_replacement.py:_preflight`/`replace_path`), when `num_tunnels is not None`, compute the CANDIDATE path's crossings + the OTHER live paths' crossings and REJECT (`return False` with a named reason) if the total exceeds `num_tunnels`. This is a pure read (no mutation), so it never touches the snapshot/rollback. Route CREATION's commit point (`path_lifecycle.finish_path_creation`/`end_path_on_station`) applies the same gate: finishing a path that would exceed the budget aborts the tentative path (mirroring the existing non-station-release abort). Mid-gesture crossing is tentative and validated only at the commit point (deterministic; graying-out unavailable crossings mid-draw is a UX refinement deferred). Refund is automatic (removing/rerouting a line reduces live crossings → `available_tunnels` rises).
- **Structured observation** — `env.py:observe`'s `"fleet"` dict gains `tunnels_total`/`tunnels_consumed`/`tunnels_available` (mirroring the `locomotives_*` triplet), None-safe for an unbounded map. The RL PIXEL observation already shows the river (GM-09b); a crossing MARKER (below) makes tunnels visible there too. Save/checkpoint are DEFERRED (river maps can't serialize until GM-09f, and a derived count restores from the path records — no `numTunnels` persistence needed this unit).
- **Rendering** — a crossing marker drawn AFTER the network (on top, unlike the terrain band which is under it): a new small pass (in `terrain_renderer` or a sibling) iterates each path's centerline crossings (reusing `crossings.py`) and draws a small tunnel glyph at each intersection point; CLASSIC (no rivers) draws nothing (byte-identical). The segment test returning the entry point feeds this.

## Invariants & risk

- **Determinism**: crossings are computed from `path_order`-independent CENTERLINES, so a crossing count depends only on the logical route, not on other lines' presence; `Mediator(seed=N, map_definition=RIVER)` and every route edit are reproducible.
- **Rollback/refund correctness (highest risk, de-risked by design)**: because `available_tunnels` is derived, the `replace_path`/`remove_path`/creation rollbacks restore it with no new snapshot field — the recon's "hook a stored counter outside the snapshot scope" trap is avoided entirely. The only new logic in the transaction is a read-only preflight gate.
- **CLASSIC/behavior-preservation**: CLASSIC's `tunnel_budget is None` → no gate, no crossings (no rivers), `available_tunnels` None — every existing route-edit/fleet test and the CLASSIC determinism locks are unaffected. No save-schema/checkpoint change; the `save-v1.json` fixture stays frozen; the fail-closed map guard stays.
- `crossings.py` stays import-safe (geometry + tuples only); the fleet-resource conservation invariant (`available + consumed == total`) holds for tunnels by construction.
