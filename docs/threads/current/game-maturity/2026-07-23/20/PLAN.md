## Scope decision (the load-bearing choice)

The roadmap: GM-09b "adds terrain/station regions and the first river map"; GM-09f owns "menu/save/high-score/RL-task integration". The second map naively triggers the deferred save-schema map field — but **GM-09b keeps the fail-closed `_require_classic_map` save guard and DEFERS the save-v2 + high-score-v2 migration to GM-09f**, because:

- The human shell (`main.build_game`) still constructs the DEFAULT Classic map; there is no menu map-selection until GM-09f, so no save-capable human path ever constructs a river `Mediator`.
- The RL path uses authenticated checkpoints, not `save_game`; `train_rl.py --map river` needs no save-schema change (the manifest already carries map identity from GM-09a2).
- Therefore no code path in GM-09b calls `serialize_game` on a river Mediator; the guard simply REJECTS a hypothetical river save (fail-closed, never a silent classic mis-save) until GM-09f lands map selection + save-v2 together, where the field is actually needed.

This is roadmap-aligned (D-032/D-033 defer the save/high-score map fields to GM-09f), avoids the high-risk frozen-`save-v1.json` migration in this unit, and keeps the CLASSIC determinism locks untouched. Recorded as a decision; the plan review must confirm it. (The GM-09a2:B STATE/EVIDENCE lines that read "must land in GM-09b" are corrected to GM-09f in this unit's delivery.)

## Design

- **The river `MapDefinition`** (`src/maps.py`, kept import-light — only `config` + `geometry.type`, so regions/river are stored as PLAIN coordinate tuples, NOT `geometry.Polygon`/shapely, which the spawn/render consumers convert): `MapDefinition` gains two additive fields defaulting to empty — `spawn_regions: tuple[tuple[tuple[float, float], ...], ...]` (polygons, as point tuples, where stations may spawn) and `rivers: tuple[tuple[tuple[float, float], ...], ...]` (obstacle bands to render, point tuples). `CLASSIC` keeps both empty (region-less, byte-identical). `RIVER = MapDefinition(map_id="river", map_definition_version=1, <CLASSIC palette>, spawn_regions=(LEFT_BANK, RIGHT_BANK), rivers=(CENTRE_BAND,))` — a single vertical river down the screen centre (x in ~[0.46, 0.54]·width) splitting the play area into two land banks; stations spawn only on the banks. Registered in `_REGISTRY` so `resolve_map("river", 1)`, `map_by_id("river")`, and `KNOWN_MAP_IDS == ("classic", "river")` expose it.
- **Region-aware spawning** (`src/entity/get_entity.py`, `src/utils.py`), strictly ADDITIVE like GM-09a's palette params: `get_random_position` gains an optional `region`/`bounds` keyword (None-sentinel → the current global `screen_width×screen_height` + `padding_ratio`, so CLASSIC's RNG draw order is byte-identical). `get_station_spawn_position`/`get_random_stations` thread an optional `spawn_regions` param; when present (RIVER), a candidate is drawn and REJECTION-sampled to fall inside a region (outside the river), preserving the 8-candidate weighted-centroid structure and the `random_source` draw for CLASSIC. `Mediator.get_initial_station_pool` threads `self.map_definition.spawn_regions` alongside the palette. Determinism: CLASSIC (empty regions) draws identically — pinned by the existing `test_gm09a_maps.py` `_CONSTRUCT_FP`/`_TRAJ_FP` and the frozen `save-v1.json` positions, which MUST NOT move.
- **River rendering** (a NEW small `src/rendering/terrain_renderer.py`, NOT growing the near-ceiling 478-line `game_renderer.py` per D-028/D-029): `draw_terrain(surface, map_definition)` paints each river band (a filled polygon/band in a water color) UNDER the network — called at the top of the render path (in `main.run_game` between the background fill and `renderer.draw`, or first in `GameRenderer.draw`), so lines and stations sit on top. A `MapDefinition` with no rivers (CLASSIC) paints nothing (byte-identical). Byte-stable and deterministic.
- **Map construction/selection**: GM-09b needs only the API path — `Mediator(map_definition=RIVER)` (region-aware spawn) and RL `--map river` (already wired). Menu/title selection stays GM-09f. The save guard stays fail-closed (a river Mediator cannot serialize).
