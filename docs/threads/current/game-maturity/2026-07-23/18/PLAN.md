## Design

The load-bearing invariant: **a map-ABSENT task descriptor must serialize byte-for-byte identically to today** (so every legacy manifest keeps its exact fingerprint), while a map-BOUND descriptor gains the map keys and an explicit version. Presence of the map keys IS the descriptor-version signal — no field is added to the legacy descriptor (which would change its bytes/hash).

- New `src/maps.py` (< 300): a small, dependency-light map-definition layer.
  - `MapDefinition` (frozen): `map_id: str`, `map_definition_version: int`, and the station-generation parameters a map controls (the config constants the recon lists: `num_stations`, `initial_num_stations`, shape lists, unique-spawn start/chance, spawn bounds/padding). `CLASSIC = MapDefinition(map_id="classic", map_definition_version=1, ...)` captures EXACTLY today's constants so its generator reproduces current spawning.
  - A `generate_initial_station_pool(map_def, context)` (or a method) that performs the current `get_initial_station_pool` pipeline parameterized by the map — same RNG draw order, so `Mediator(seed=N)` under `CLASSIC` yields byte-identical stations to today (pinned by a determinism test against the pre-change behavior).
  - `map_by_id(map_id) -> MapDefinition` lookup; `CLASSIC` is the only entry this unit. `maps.py` imports only geometry/config-level constants (no gameplay/rendering/save), so it stays off headless-forbidden sets by construction and is import-safe for RL.
  - `Mediator` selects a `MapDefinition` (default `CLASSIC`) and routes initial spawning through it — behavior-preserving (the default path is byte-identical; a determinism test proves it against a clean pre-change worktree).
- Versioned task-descriptor identity (`src/rl/protocol.py`):
  - `TaskSpec` gains `map_id: str | None = None` and `map_definition_version: int | None = None` (default `None` = legacy/map-absent). A helper normalizes "Classic" to `("classic", 1)` when the CLI/env opts in.
  - `task_descriptor(spec)`: when `map_id is None`, emit the CURRENT flat dict UNCHANGED (byte-identical → legacy hash). When `map_id` is set, add `mapId`, `mapDefinitionVersion`, and an explicit `descriptorVersion: 2` (legacy is implicit v1 by absence). `task_fingerprint` is unchanged (hash of the canonical descriptor).
  - A sensitivity test asserts changing `map_id` changes the fingerprint, and that a map-absent spec's descriptor/bytes are IDENTICAL to the pre-change output (regression-locked against the recorded legacy descriptor + the real fixture hash `c2ef342f…`).
- Threading (additive, map-keys only on the new version):
  - `manifest_schema.py`: a new **v3** (`...-training-manifest-v3`) that is `_V2_KEYS | {mapId, mapDefinitionVersion}`; v1/v2 remain exactly map-free. `TrainingManifest` carries optional `map_id`/`map_definition_version`; `to_dict`/`from_dict` emit/accept them only for v3. `create_training_manifest` writes v3 with map fields when a map-bound spec is used, else keeps writing the current version map-free.
  - `player_env.py` / `env.py`: `TaskSpec` construction threads the selected map; `Mediator(seed=...)` is built under the map; `_info`/observation gain no map field this unit (observation map identity is GM-09f). `training.py` `PlayerEnvThunk`/`make_env_thunks` carry `map_id`/`map_definition_version` so subprocess workers rebuild the same map.
  - `task_spec_from_manifest` (`training.py`): read map fields when present (v3) → map-bound spec; ABSENT (v1/v2) → map-less spec (`map_id=None`) → legacy descriptor → legacy hash. This is the compatibility guarantee; a test drives the real recurrent-final-smoke v1 manifest through it and asserts the reconstructed fingerprint equals the stored `c2ef342f…`.
  - `scripts/train_rl.py` / `evaluate_rl.py`: a `--map` arg defaulting to `classic` (train writes a v3 map-bound manifest; resume/evaluate reconstruct via `task_spec_from_manifest`, so a legacy manifest still validates). No behavior change when `--map classic` vs the legacy default is chosen is documented (they produce DIFFERENT fingerprints — a new Classic run is a new descriptor version, which is intended; legacy manifests keep the old hash).
- High-score: `main.run_game`'s `record_score` sources `map` from the active `MapDefinition.map_id` (Classic → `"classic"`, the exact existing wire value) instead of the hardcoded literal, so existing `saves/highscores.json` boards stay keyed identically. `HIGHSCORES_MAP_CLASSIC` stays `"classic"` as the Classic id.
- **Deferred (documented scope decision, flagged for the plan review):** the SAVE-schema map field. With only the Classic map, a save's map is unambiguous (always Classic) and a save-schema v2 + a new frozen fixture would add an inert field for zero current behavioral benefit while risking the byte-frozen `save-v1.json`. D-026 reserves map for the version that "synthesizes classic for older saves" — that naturally lands with the FIRST alternate map (GM-09b) or the GM-09f save/menu integration, where the field is meaningful. GM-09a keeps `save-v1` byte-identical and records no map. (This is the one deviation from the roadmap's literal GM-09a surface list; the plan review should confirm it.)

## Invariants & risk

- **Legacy byte-compat (highest risk):** a map-absent descriptor is byte-identical to today; the real recurrent-final-smoke v1 manifest reconstructs to `c2ef342f…`; v1/v2 manifests stay exact-key-valid; `save-v1.json` stays byte-frozen; the high-score `"classic"` wire value is unchanged.
- **Determinism:** `Mediator(seed=N)` under `CLASSIC` yields byte-identical stations/trajectory to the pre-change code (proven against a clean pre-change worktree, the GM-08a byte-identity pattern).
- No checkpoint-schema change (observation/checkpoint map identity is a later unit), no rendering change, no save-schema change this unit; the frozen v1-v5 checkpoint and save-v1 fixtures stay byte-exact.
- `maps.py` imports no gameplay/rendering/save module; RL imports it transitively via the env — a determinism/isolation test confirms no new headless-forbidden import and that `maps` is import-safe.
